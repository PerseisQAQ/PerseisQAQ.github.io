引入

让我们先来看一道快速幂的模板题：Problem - 2035 (hdu.edu.cn)

题面：

求A^B的最后三位数表示的整数。
说明：A^B的含义是“A的B次方”

Input：

输入数据包含多个测试实例，每个实例占一行，由两个正整数A和B组成（1<=A,B<=10000），如果A=0, B=0，则表示输入数据的结束，不做处理。

Output：

对于每个测试实例，请输出A^B的最后三位表示的整数，每个输出占一行。

相信大家在初学程序设计语言时应该都练习过求幂，只要写一个简单的循环就能轻松完成。

大概代码是这样的：

但是，当我们求解2^100时，计算出来的结果却为0 TAT

这是因为int所能存储的最大值为2147483647，即2^31 -1，很明显我们的结果超出了这个数，所以发生了溢出。因此，题目会要求我们对结果取模。

因此我们来了解一下取模运算法则qwq：

  • (a + b) % p = (a % p + b % p) % p
  • (a - b) % p = (a % p - b % p ) % p
  • (a * b) % p = (a % p * b % p) % p

在这里，我们需要用到的是第三条式子awa。由这个式子可以得知，只要我们在循环的每一步中都进行取模，也可以计算出我们想要的结果！！

但是当我们分析一下，会发现这个算法的时间复杂度是O(n)，n为指数。在计算一个比较大的数时就会很不给力了QAQ，所以我们有必要了解一种更加快速的算法。

快速幂

众所周知，2^6 = 4^3 OAO

所以快速幂的本质就是每次将指数减少一半，再将底数平方，这样原本要跑10000次循环的计算就只用跑5000次，是一个很大的提升。

注意到，当指数为奇数时，我们不能简单地将指数减少一半。而再通过观察，可以发现：

4^3 = 4^2 * 4

所以，我们单独拿出一个4，指数又能继续缩小了，这样的效率就比之前高了很多。
