T1 题目没按难易排

有趣的题面：

众所周知，贼龙有着“大胃怪物王”的称号，当他饿肚子的时候会变得异常暴躁，有事甚至会攻击那些远比自己强大的怪物，在捕杀了草食龙后，贼龙会将其一口气尸体整个吞下去，此时贼龙的腹部会十分夸张的涨大，让人不由得想起冰海里的化鲛。在捕猎成功后，贼龙会回到自己的领地，把捕获到的猎物吐到地上，然后唤来小贼龙们进食。

每条贼龙要哺育的小贼龙的数量各不相同，每条小贼龙都只需要进食一份肉块，古代树森林里也生活着体型各不相同的草食龙，分别为大型，中型，小型，分别能分成10、8、6块，贼龙吞下他们分别需要25、20、15分钟。(贼龙肚皮贼大，能吞好几只一起带回去)

如果贼龙需要喂养n条小贼龙，那么你的任务是确定贼龙吞下的草食龙至少能分成n份肉块所需要的最小分钟数。

如果有12条小贼龙需要喂养，贼龙就必须带回至少能分成12块的草食龙，他可以带回2头小型的，正好12份，需要30分钟。

如果有15条小贼龙需要喂养，贼龙就必须带回至少能分成15块的草食龙，他可以带回1头小型的和1头大型的，共16份，需要40分钟。

Input

第一行包含一个整数 t (1≤t≤10^4) 表示测试用例的数量

每个测试用例由一行组成，其中包含一个整数n (1≤n≤10^16) — 需要喂养的小贼龙的数量

Output

对于每个测试用例，打印一个整数——捕捉总共至少包含n份的草食龙所需的最小分钟数。

原题链接：Problem - 1555A - Codeforces

  • 思路：

    我们先来列一下已知条件：

    • 大型草食龙 吞食需要25分钟 可以分成10块肉
    • 中型草食龙 吞食需要20分钟 可以分成8块肉
    • 小型草食龙 吞食需要15分钟 可以分成6块肉

    我们不难发现 25 / 10 = 20 / 8 = 25 / 6 = 2.5

    ​ -> 也就是说，最理想的结果就是将 n * 2.5

    但仔细想想就会发现，并不是每个 n 都可以刚好由10、6、8凑出，因此会有如下分析：

    • 易知当n为奇数时，肯定是凑不出来的OvO，所以应把 n + 1 凑成最近的不小于它的整数

    • 此时我们来讨论偶数：

      • 已知即使是小型草食龙也有6块肉，所以当 n < 6 时，我们应输出最小值 15

      • 之后我们便可将问题转化为： 6 x + 8 y + 10 z = n

      • 再将其转化为：3 x + 4 y + 5 z = n / 2

      • 我们再观察一下，当 n >= 6 ，也就是 n / 2 >= 3 时，可以得到以下式子：

        n / 2 = 3 = 3

        n / 2 = 4 = 4

        n / 2 = 5 = 5

        n / 2 = 6 = 3 + 3

        n / 2 = 7 = 3 + 4

        n / 2 = 8 = 3 + 5

        n / 2 = 9 = 6 + 3

        ……

        以此类推

        由此得到：任意大于等于3的数都可以由 3，4，5拼凑出

        -> 任意大于等于6的偶数都可以由6 ，8，10拼凑出

        至此，题目已解完

T2 不要纠结在一题

有趣的题面：

yyj很喜欢数字，某天他有两个正整数a,b (1≤a≤1e6, 1≤b≤1e6 (1000000)) 他定义： 1.能被a*b整除的数为“yyjyyds” 2.不能被a*b整除但是能被a整除的数为 “yyjak” 比如有a=5，b=3，则15和 30是“yyjyyds” 而10和 50 是”yyjak” 但是16, 7都不符合. 这时yyj问你 ：找出 3 个不同的正整数 x、y 和 z，使得其中一个是”yyjyyds” ,另外 2 个是“yyjak”，并且满足 x+y=z。

Input

第一行包含一个整数 t (1≤t≤10000)——测试用例的数量。 每个测试用例的第一行包含两个整数 a 和 b (1≤a≤1e6, 1≤b≤1e6)。（注意数据范围）

Output

对于每个测试用例打印： “YES”和 3 个不同的正整数 x、y 和 z (1≤x,y,z≤1e18 (1后面18个0 )，使得其中一个是”yyjyyds”，其他 2 个是”yyjak”，并且 x+y=z。 如果不存在答案，则为“NO”。（yes,no不区分大小写） 如果有多个答案，打印任何一个。

原题链接：Problem - 1521A - Codeforces

wxyyyds!!

  • 思路：

    易知“yyjyyds”与“yyjak”都是a的倍数

    我们首先来找一找不存在答案的情况，即无法凑出2个“yyjak”与1个“yyjyyds”的情况

    ​ ->思考得知，当b = 1时，只要x、y、z想要成为a的倍数，就一定会成为b的倍数

    ​ -> a * b = a ，即一定会成为“yyjyyds”，无法满足2个“yyjak”

    于是当 b = 1 时，答案不存在

    接下来我们来找一找答案

    • 我比赛时的思路

      x，y，z皆为正整数且为a的倍数，其中“yyjyyds”为a*b的倍数

      很容易能发现出一组构造方案： x = a，y = a * ( b - 1 )，z = a * b

      但是但是！！这里面是有bug的！

      当b = 2时 会变为 a，a，2 * a的情况

      因此我们可以对b进行特判，若b = 2则输出x = a，y = 3 * a，z = 4 * a

    • 事实上

      构造 x = a，y = a * b ，z = a * ( b + 1) 就好了

      我知道我很菜了别骂了别骂了孩子傻了呜呜呜

T3 可能下道更简单

有趣的题面：

yyj喜欢玩搭积木，她喜欢把积木搭成金字塔的样子，但是她又有强迫症，每层积木都要搭成她想要的样子，不然她就不和你玩了，（从上往下数）第一层要搭一个，第二层要搭1+2=3个，第三层要搭1+2+3=6个，第i层要 1+2+3+……+i 个，现在呢，你手上有n个积木，要搭一个最高的积木给yyj看，来获得她的芳心，那么，思考一下，可以搭多少层积木呢

Input

输入一个数字n(1<=n<=10^4),是你拥有的积木数量

Output

输出一个数字，表示你可以最多搭几层

原题链接：Problem - 492A - Codeforces

别看title，我信你个鬼

  • 思路：

    是个模拟呢

    已知搭第 i 层需要1 + 2 + …… + i个积木，因此我们可以先用一个数组存下搭建每一层所需要的积木数量

    看了看数据范围 1 <= n <= 1e4

    嗯，500肯定够了（多太多了啊喂）（1+2+…+500 = 125250）

    然后我们再模拟每一层搭建，如果搭建到目前这一层需要的总数量超过了拥有的积木的数量，则运行结束，这一层无法搭建

刘成富是一个性格开朗、热情的男孩子。他的梦想是每天都能快乐地唱歌，并且打maimai。有一天，在练歌的时候，富同学无意中发现有一些按键可以让歌曲变得更好通关。他把这些词叫做 “AKmaimai”。 经过多次尝试，他发现在每首歌中，总共有n个 “AKmaimai”。她想知道每一个 “AKmaimai “在他的歌曲中最多有多少次不相干的出现。 注意：一次出现意味着AKmaimai是歌曲中某一位置的连续子串。 举例来说。abababa “中 “aba “的最大不连续出现次数是2。

Output

对于每个测试用例，输出n行，每行一个整数，表示答案。

在远古，有k条邪恶的时间龙统治着acm村庄。终于，年轻的wxy勇士组织村民反抗这些邪恶的时间龙。 wxy将军把n个村民分成m种兵种，每种兵种能组成一种阵法抵御时间龙。每击杀一条时间龙，这个兵种里面所有的村民将失去一点生命值，当有村民生命值为负数时，这个村民将反叛，从而导致这个兵种将无法组成阵法（只有阵法能击杀时间龙）。 wxy忙于击杀时间龙，请你求出在wxy领导下，最多击杀多少时间龙。

T10 欢迎大家来到实验室！

不怎么有趣的题面：

给定一个正整数n，问是否存在一个奇数a(a>1)，且n%a=0。如果存在存在这个奇数，那么就称正整数n为神奇数。给定一个正整数n，如果它是神奇数，输出YES，否则输出NO。

Input

首先输入一个正整数t，然后t组案例，然后每组案例输入一个正整数n(2<=n<=10^18)。

Output

如果n是神奇数，输出YES，否则输出NO。注意应该有t组输出。

原题链接：Problem - 1475A - Codeforces

关于我看到这题就开始暴力枚举每个比n小的奇数结果在本地连样例都跑不出来这件事

  • 思路：

    思维题（咬牙切齿）

    首先，1不可以是神奇数，因为a>1 ，1不能被大于它的数整除

    在这个基础上，其余的奇数都是神奇数，因为他们可以被本身整除

    再来看偶数(我一开始很傻的把偶数都判成了NO)

    我们可以发现6，12等也是可以被奇数整除的

    所以我们可以将n一直除以2，直到n成为一个奇数

    • 如果此时的n不是1，那么他原本是个神奇数
    • 如果此时n = 1，说明他原本不能被1以外的奇数整除，不是个神奇数

rick会去问kevin一共m个问题，每次问题给出一个区间[L,R]，如果rick想要这个句子的这个区间内不存在回文子串，那么kevin要去修改多少个这个区间的字符，才可以要这个区间不存在回文子串。

Input

第一行给出n,m表示字符串长度和查询次数。1≤n,m≤2⋅1e5

第二行给出字符串S

接下来m行每行一个L与R，表示询问区间的左右端点 (1≤L≤R≤n)

Output

每个询问输出一个数字表示，kevin要去修改多少个这个区间的字符，才可以要这个区间不存在回文子串。
