T1 Yuuki的魔法禁书目录

有趣的题面：

Yuuki得到了一本名为index的魔法书，据说里面记载了十万三千种魔法，只要把整本书读懂就能自行学会所有魔法，这本书有n个章节，每个章节都被一个特定的前提章节列表封印住了（即要读懂该章节，必须先把所有前提章节读完才能读）

可是Yuuki现在一个字也看不懂，他只能从头到尾一遍一遍的读，请你帮他算一算他要读几遍才能读懂这本书，又或者完全不可能读懂全部章节。

Input

有t (1<=t<=2e4)组测试样例

每个测试样例的第一行是一个整数n（1<=n<=2e5)—-表示章节数目

接下来的n行，每行以一个k（0<=k<=n-1)开始—-表示第i章前提章节的数目，再输入k个前提章节的编号

Output

对于每个测试样例，输出Yuuki最少要读几遍书才能读懂全书，如果不能读懂，输出-1

原题链接：Problem - 1573C - Codeforces

  • 思路：

    拓扑排序，还不太会，等会update

T2 无比强大的nero

有趣的题面：

Yuuki很喜欢玩音游，但是他玩的非常菜，技术怎么也提升不上来，这天他找到acm组里最强的nero想要看看自己和他的差距有多大，于是两人便决定在n首歌曲中每人挑一首打

可是Yuuki实在是太菜了，导致他每次只能选择难度最低的一首歌，而强大无比的nero每次都会选择最难的一首歌

Yuuki想知道他和nero的差距有多大，你能帮他算一算吗（他们的实力差距是所选歌曲的难度差）

Input

一共两行

第一行为歌曲的数量n(1≤n≤1000)。

第二行为n首歌曲的难度，之间以一个空格分隔 (每首歌的难度保证为非负整数）

Output

输出一行，表示Yuuki与nero的实力差距

原题链接：整数序列的元素最大跨度值 - 题库 - 计蒜客 (jisuanke.com)

  • 思路：

    走一遍循环找到最大值与最小值，然后作差

T3 jel的上分之旅

有趣的题面：

近些时日，jel沉迷于酒馆战棋，但是连着下了几天棋，他的分数也不见涨，因此他非常生气，觉得一定是发牌员Bob的问题，于是他决定给自己加一项“+=”技能来使得他的随从更加强大以击败对手

jel的随从有两种属性，生命值a和攻击值b，而jel每一次使用“+=”技能可以选择下面其中一种效果：

\1. a+=b

\2. b+=a

现在已知对手的血量是n，他想知道最少要使用多少次“+=”技能可以击败对手（击败条件为a或b任意一种属性大于n）

Input

第一行输入一个整数T，为T个测试样例。（1<=T<=100）

之后每个样例输入三个数a，b，n。（1<=a,b<=n<=1e9）

Output

输出每个样例的最少技能使用次数

原题链接：Problem - 1368A - Codeforces

  • 思路：

    • 当a<=b时，a+=b
    • 当b<a时，b+=a
    • 当a或b有一方大于等于n时，输出次数

T4 yyj的衣柜

有趣的题面：

今天是yyj要去约会的日子，为了显示出他的帅气，他决定穿着编号为n的衣服出门，可是他忘了自己的衣服放在哪里了，yyj家是一栋可以无限增长的高楼，每层都有他的衣柜，除了第一层有两个衣柜，其他每层均有x个衣柜，且衣柜的编号依次递增，yyj的每一件衣服都是独特的，所以他的每一个衣柜中只会放一件衣服，请你帮助yyj找到他的衣服在哪一层

Input

输入的第一行包含一个整数t(1≤t≤1000)——测试用例的数量。

然后是t行测试用例。 每行包含两个整数n和x(1≤n,x≤1000)–yyj要去取的n号衣服和除第一层以外的每层的衣柜数(一层只有两个衣柜)。

Output

输出yyj要前往的层数

原题链接：Problem - 1426A - Codeforces

  • 思路：

    • 特判当n<=2时，yyj要去第一层
    • 让n=n-2，与x做取余运算
      • 如果n能被x整除，yyj要去n/x+1层
      • 如果n不能被x整除，yyj要去n/x+2层

T6 BBBB

不有趣的题面：

输入一个正整数 n，判断 n是否是质数，若 n是质数，输出"Yes",否则输出"No"。

输入一个正整数 n(n≤2×10^7)。

Output

如果 n 是素数输出"Yes"，否则输出"No"。输出占一行。

T7 CCCC

不有趣的题面：

从1-n中选出m个数，要求同样的数字不能重复选择，按照字典序正序输出所有方案。

例如：从1到4中选出2个数，共有6种方法，按照字典序输出，依次为：

1 2
1 3
1 4
2 3
2 4
3 4

Input

输入共2个数n, m，中间用空格分隔。（1 <= m <= n <= 20）

Output

按照字典序正序输出所有方案。

    用step记录当前保存了多少个数，用a数组存目前选取了的数

T8 饥饿的yyj

有趣的题面：

wxy饿坏了，饿到能一口吃一个新生，心情不好的时候一口吃两个！现在有N个新生，wxy想知道他有多少种吃完新生的方案。例如N=1，wxy一口就吃掉他，共一种方案。若N=2，则wxy可以第一口吃一个，第二口吃另一个，或者直接一口吃两个共两种方案。若N=3，那么wxy可以第一口吃一个剩两个，也可以第一口吃两个剩一个，共2+1种方案。现在给出N，请你求出wxy吃完新生的方案数，如果答案正确，那么该新生就可以免于被吃的命运。
